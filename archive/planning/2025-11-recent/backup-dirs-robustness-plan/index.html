    Critical Issues to Fix¶

    1. BLOCKING: zstd Compression Broken (Lines 642-664)¶

    Problem: - File descriptor manipulation creates corrupted archives - tar output leaks to stdout - Archive contains only filenames, not actual data - Current broken code: tar -c ... 2>&1 >&3 3>&- | zstd

    Root Cause: - Attempting to separate tar's data stream (stdout) from verbose output (stderr) while piping - File descriptor gymnastics 2>&1 >&3 3>&- not working correctly - tar's archive data and verbose listing getting mixed

    Solution Approaches to Try (in order):

    1. Named Pipe (FIFO) Approach - Most robust 

      # Create named pipe for verbose output
mkfifo /tmp/tar_verbose_$$
# Read verbose output in background for counting
cat /tmp/tar_verbose_$$ | { count_loop } &
# Pipe tar data through zstd, send verbose to FIFO
tar -c ... -v ... 2>/tmp/tar_verbose_$$ | zstd > "$backup_dest"

    2. Process Substitution Approach - Cleaner 

      # Use process substitution for verbose counting
tar -c ... -v ... 2> >(count_loop) | zstd > "$backup_dest"

    3. Explicit File Redirection - Most compatible 

      # Redirect verbose to temp file, tail -f for counting
tar -c ... -v ... 2>/tmp/tar_verbose_$$ | zstd > "$backup_dest" &
tail -f /tmp/tar_verbose_$$ | { count_loop }

    4. Dual Pipeline - Separate concerns 

      # Create tar without compression, capture verbose
tar -cv ... 2>&1 | tee >(count_loop) >/dev/null
# Then compress separately (if needed)

    5. Simplify - No Real-Time Count - Fallback only if all else fails

    6. Show spinner without count during compression
    7. Extract and count entries after completion
    8. Not ideal but better than broken

    2. Code Quality Issue: Unused Variable (Line 55)¶

    Problem: - PROGRESS_PADDING_PERCENT=100 is unused after removing padding logic - Triggers shellcheck warning

    Fix: - Remove line 55 entirely

    Implementation Results¶

    Issues Fixed¶

    1. zstd Compression (CRITICAL)
    2. Problem: File descriptor manipulation was creating corrupted archives
    3. Solution: Use grouped command { tar | zstd; } 2>&1 | { count } to properly separate archive data from verbose output
    4. Research: Based on Unix Stack Exchange solution for tar piping

    5. Result: Creates valid .tar.zst archives, fully working multi-threaded compression

    6. Exit Code Handling

    7. Problem: Script returned exit code 1 even on success
    8. Root Cause: cleanup trap's kill command returning 1 when process doesn't exist
    9. Solution: Added || true to kill commands and explicit return 0 in cleanup function

    10. Result: Proper exit code 0 on success, exit code 1 on failure

    11. Cross-Platform Compatibility

    12. Problem: GNU stat --format='%s' returns exit code 1 on macOS despite working correctly
    13. Solution: Replaced with universal wc -c < file command

    14. Result: Works reliably on macOS and Linux

    15. Code Quality

    16. Problem: Unused PROGRESS_PADDING_PERCENT variable
    17. Solution: Removed unused constant
    18. Result: Shellcheck passes with zero warnings
